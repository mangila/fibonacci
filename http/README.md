### Testing SSE with `curl`
To observe the real-time stream via `curl`, use the `--no-buffer` (`-N`) flag. This prevents `curl` from buffering the output, allowing you to see events as they arrive:

```bash
curl -N "http://localhost:8080/api/v1/sse/livestream?streamKey=test-session"
```

### The `streamKey` Parameter
Most streaming endpoints require a `streamKey`. This is a unique identifier (typically a UUID in production) used by the backend to manage your SSE session and heartbeats. For manual testing, any unique string will suffice.

### HTTP/2 Benefits
While the standard SSE protocol works over HTTP/1.1, using H2C (via `h2c.sh`) allows for multiplexing and more efficient header compression, which is recommended for high-throughput streaming applications.
